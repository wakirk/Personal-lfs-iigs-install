menu_system.lib specifications:
	Is not a stand alone script, meant to be included in another file for execution.

Usage:
	Inclusion in host script:	
	source menu_system.lib   This includes the menu system into the host script.

Files:
File 1: Menu Data File
					The exact name is what ever is passed to menu_setup().

TSV file			Purpose:   This is where the menu items, actions, and function names are at.

Foramt:
	Function_name<tab>file_Name<tab>Display_Name

Format Descriptor:
	Function_name:		Is the name of the function in the script to call if the entry is selected.  Must be present.
	File_name 			Is the name of the file to be called if required. (can be blank, i.e. ...<tab><tab>...)
	Display Name 		Everything else is the display name to EOL. That gives me the most freedom to make it what I want.

Can have as many items as needed, they will be assgined to menu items in Sequence from top to bottom.

This file is not edited by the script.

File 2: Save File
				Purpose: Keeps track of the status of the menu flags, feature presistance.

				fields:
						Function Name: 	Text: Used to keep track of each menu items's status by the function name
										used in the TSV, as that field is both required and short.
						seporator:		(leaving this up to AI steps, but tab recommended, comma acceptable)
						Status Code:	Character:  Code to use to set the menu's status code.  

Reason for Menu and Save file seporation:
				Ease of Reset, I just delete the save file and everything is cleared. Useful for a script to wipe them all
				allowing for a global reset while leaving the menu intact.

Matching:		the Function name field in the .TSV needs to match the function name field in the .sav file. 
				this function name field is required in both, and also it should be present in the script, if 
				it is not found in the script, stop everything with an error:
				"Menu System Load Error: Funciton [function name] is missing in the host, please correct and restart."
				So we can safely assume it's always there now.
				

Functions:

menu_setup ()	paramter: TSV filename.  
				Pass it the data file name with the entries to display.  Calling it again should be
				an error.  It should load the data file and prep any variables data.  This gives the
				system a chance to prep for all startup and needed Tasks.

				Return Code:
					0:  Locked and Loaded.
					1:  Already Loaded. 	Don't do this more then once.
					2:  File Error			Something went wrong with the TSV file its self.
					3:	Format Error		Something went wrong with the data.
					4:	... any other errors needed.

menu_save()		Parameters:		None.
				Save the status of the menu entries filename: ($HERE/.menu.sav)
				During save, only use what currently active in memory, so it's written fresh to accomondate changes.
				Return Code:
					0 		Success
					1		File Related Error

menu_load()		Parameters:		None.
				Load the status of the menu entries filename: ($HERE/.menu.sav)
				During Load, try to match function Name in save file to what was loaded form .TSV file.
				* If a menu matches on function name, set that fields status as what is in save file.
				* if save's field is not found in the menu data, assume the entry was deleted and ignore it.
				* if the menu has an entry that's not in the save file, defualt it to Incomplete. (new menu entry)
				That should clear up any version changes.
				Return Code:
					0 		Success
					1		File Related Error
					2		Save File Format Error
					3		TSV was not prevously loaded.
							Halt with Error: "Error.  Load Menu Data File First."

Run_Menu()		Paramters:	None
				Main menu display, this controls the menu, callbacks, and control loop
				Step 1:
					Any setup or init that is needed.
				Step 2: 
					Main Menu Loop 
					step L1:
						pre-menu display
						Check for Menu_Pre_Render() callback, 
						if found, all it, and assume it setup the screen.
						if not found, just clear the screen.
					step L2:
						Menu Entries Dispaly
						Show the list of menu items (see extended details)
					step L3:
						post-menu dispaly
						Check for Menu_Post_Render() callback,
						if found, call it
							if it returned 0 do nothing.
							if it returned 1 show the menu instructions
						if not found, show the menu instructions
					step L4 
						Get keystroke
						Process keystroke
						If Menu_Select_*() exists, call it, otherwise keep going.
						* matches the letter or number of the menu choice.
						Call funtion_name is deisgnated in .TSV file.
					step L5, if menu paging is requried, update the list
					step L6, Process remaining commands
					:end Main Menu loop
				Loop Exited
				Step 3 Any Post Menu Cleanup, likely about to save and/or exit.

for Step L2 display layout
